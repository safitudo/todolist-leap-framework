# todolist-leap

> The original [LEAP](https://github.com/safitudo/leap) proof of concept. A todo list web application where code is a commodity.

## What is LEAP?

**L**LM **E**ngineered **A**pplication **P**attern. A development philosophy where apps are designed as:

- **Prompts** — declarative intent (what to build)
- **Schemas** — contracts between parts (the architecture)
- **Tests** — behavioral specifications (the definition of "correct")

Code is generated on demand by any AI agent. It's not committed. It lives in `src/`, which is gitignored.

**There is no application code in this repository.**

## What's in this repo

```
master.md         ← read this first (the AI does too)
schemas/          ← contracts between parts (human-authored)
parts/            ← prompts + schemas for each component
tests/            ← behavioral tests (human-authored guardrails)
src/              ← GITIGNORED — generated by AI
```

## How to use

1. Clone this repo
2. Install the [leap-skill](https://github.com/safitudo/leap-skill) Claude Code plugin
3. Say: **"build the app, start from master.md"**
4. The agent generates the app into `src/`
5. Run `npm install && npm test` to verify
6. Run `npm run serve` to launch the app (picks a random port)

That's it. The AI is the compiler. The prompts are the source code.

## Why this matters

The internet is benchmarking AI on "how well does it write code." But if code is a commodity, that's like benchmarking a printer on paper quality.

The real questions are:

- How good are your **schemas** at defining boundaries?
- How good are your **tests** at catching when generated code is wrong?
- How good are your **prompts** at conveying intent?

Value shifts from **writing code** to **specifying behavior**. Tests aren't a safety net — they're the product.
